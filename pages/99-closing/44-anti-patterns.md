# Анти-патерни — чого уникати

</div>

<div class="lang-en">

# Anti-patterns — what to avoid

</div>

<v-clicks>

- <span class="lang-uk">**Dump усього репо** в кожен запит — шум, втрата фокусу, дорого</span><span class="lang-en">**Dumping the whole repo** into every request — noise, lost focus, expensive</span>
- <span class="lang-uk">**Приймати код наосліп** без рев'ю — баги й вразливості проникають тихо</span><span class="lang-en">**Accepting code blindly** without review — bugs and vulns sneak in</span>
- <span class="lang-uk">**Найдорожча модель на все** — спалювання грошей на рутині</span><span class="lang-en">**The priciest model for everything** — burning money on routine</span>
- <span class="lang-uk">**Vibe coding у проді** без плану й перевірки</span><span class="lang-en">**Vibe coding in production** with no plan or checks</span>
- <span class="lang-uk">**Секрети в промпт/контекст** — найдорожча помилка з усіх</span><span class="lang-en">**Secrets in the prompt/context** — the most expensive mistake of all</span>
- <span class="lang-uk">**Agent одразу** на складній задачі замість Plan</span><span class="lang-en">**Agent immediately** on a hard task instead of Plan</span>

</v-clicks>
